1. Header Inclusion & Library Linking

We begin by importing <winsock2.h>, the core API for Windows networking. The directive #pragma comment(lib, "ws2_32.lib") is crucialâ€”it tells the linker to include the actual implementation code for sockets, preventing "undefined reference" errors during compilation.

2. Socket Instantiation (The 'Phone')

The line socket(AF_INET, SOCK_STREAM, IPPROTO_TCP) creates the endpoint.
- AF_INET: Declares we are using IPv4 addresses (e.g., 127.0.0.1).
- SOCK_STREAM: Selects TCP protocol, ensuring data arrives in order and without errors, unlike UDP.

3. Binding & Listening (The 'SIM Card')

Before receiving calls, we must assign a phone number. bind() attaches our socket to Port 8080. We use htons(8080) to convert the port number to "Network Byte Order" (Big Endian) so the internet can understand it. Finally, listen() puts the socket in a passive mode, ready to accept incoming connection requests.

4. Request Parsing logic (The 'Brain')

When recv() captures the raw bytes: "GET /about.html HTTP/1.1...", we convert it to a std::string. We then use string manipulation (find() and substr()) to slice out exactly "/about.html". This logic is the heart of the server routing mechanism.

5. Response Construction (The 'Reply')

We don't just send the file; we must follow protocol. We first construct an HTTP Header containing status codes (200 OK) and metadata (Content-Length). Only after sending this header do we stream the actual binary data of the HTML file using send().
