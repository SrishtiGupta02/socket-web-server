Scalability Roadmap

Multi-Threading & Concurrency

Currently, the server is 'Iterative' (one by one). The next phase involves implementing std::thread or thread pools to handle thousands of concurrent connections without blocking, making it scalable for high-traffic environments.

Encryption (SSL/TLS)

Integrating OpenSSL libraries to facilitate the TLS handshake. This would encrypt the data stream, upgrading the protocol from HTTP to HTTPS and securing user data against interception.

Dynamic Content (CGI/FastCGI)

Moving beyond static files by adding a Common Gateway Interface (CGI). This allows the server to execute backend scripts (Python, PHP) to generate personalized HTML content on-the-fly.

Containerization (Docker)

Packaging the application into a Docker container. This ensures the server runs consistently across any environment (Dev, Test, Prod) without dependency issues, a standard practice in modern DevOps.
