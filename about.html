System Architecture

01

Initialize

The server asks the OS for a communication endpoint using socket(). It then 'Binds' this endpoint to Port 8080 on the localhost, effectively staking a claim on that digital address.

02

Listen Mode

The system enters a blocking state using listen(). It creates a backlog queue and actively monitors for incoming TCP connection SYN packets from any client browser.

03

Data Parsing

Upon a successful 3-way handshake, the connection is accepted. The server reads the raw byte stream into a buffer and parses the HTTP Header to extract the specific resource path (e.g., GET /index.html).

04

Transmission

The requested file is loaded from the disk into memory. The server constructs a compliant HTTP/1.1 Response Header (Status 200) and flushes the data stream back through the socket to the client.
